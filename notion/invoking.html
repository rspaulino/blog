To test your endpoints with Dapr, simply expose any HTTP endpoint POST GET.
Note: To choose a non-default components folder, use the --components-path option.
dapr invokeGet
Issue HTTP GET to Dapr app
Usage
dapr invokeGet [flags]
Bash
Flags
| Name | Description | | ------------ | -------------------- | | --app-id, -a | The app ID to invoke | | --help, -h | Help for invokeGet | | --method, -m | The method to invoke |
Markdown
Examples
Complex Object default ROUTE
[HttpGet] public IEnumerable<WeatherForecast> Get() { var rng = new Random(); return Enumerable.Range(1, 5).Select(index => new WeatherForecast { Date = DateTime.Now.AddDays(index), TemperatureC = rng.Next(-20, 55), Summary = Summaries[rng.Next(Summaries.Length)] }) .ToArray(); }

dapr invokePost
Issue HTTP POST to Dapr app with an optional payload
Usage
dapr invokePost [flags]
Bash
Flags
| Name | Description | | ------------- | ------------------------- | | --app-id, -a | The app ID to invoke | | --help, -h | Help for invokeGet | | --method, -m | The method to invoke | | --payload, -p | (optional) a json payload |
Markdown
Examples
Complex Object default POST
[HttpPost] public ActionResult<WeatherForecast> PostWeatherForecast(WeatherForecast weatherForecast) { Console.WriteLine($"was received {weatherForecast.Summary}"); return Ok($"Success {weatherForecast.Summary}"); }
C#
dapr invokePost --app-id api --method weatherforecast --payload '{"Date":"2009-02-15T00:00:00Z","TemperatureC":70, "Summary": "Hello" }'
