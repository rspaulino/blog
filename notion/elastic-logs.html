Running Filebeat on Kubernetes
Download the manifest file:
curl -LJO https://raw.githubusercontent.com/elastic/beats/7.8/deploy/kubernetes/filebeat-kubernetes.yaml
Bash
Everything is deployed under the kube-system namespace by default.
Settings
Change the following parameters in the manifest file:
- name: ELASTICSEARCH_HOST value: elasticsearch - name: ELASTICSEARCH_PORT value: "9200" - name: ELASTICSEARCH_USERNAME value: elastic - name: ELASTICSEARCH_PASSWORD value: changeme
YAML
Once open, go to env: section in the file. We will be changing ELASTICSEARCH_HOST and ELASTICSEARCH_PASSWORD values here.
Change ELASTICSEARCH_HOST value from elasticsearch to https://quickstart-es-http.default.svc.cluster.local and ELASTICSEARCH_PASSWORD with the one we will decoded.
kubectl get secret quickstart-es-elastic-user -o=jsonpath='{.data.elastic}' | base64 --decode; echo
Bash
- name: ELASTICSEARCH_HOST value: https://quickstart-es-http.default.svc.cluster.local - name: ELASTICSEARCH_PORT value: "9200" - name: ELASTICSEARCH_USERNAME value: elastic - name: ELASTICSEARCH_PASSWORD value: hmcsjjj7bkvk4s4k1rsxzd5t
YAML
We'd be skipping how to work with certificates in this quickstart article. Let's disable SSL verification (bad idea for production!). Go to output.elasticsearch section while in filebeat-kubernetes.yaml file to add ssl.verification_mode: none
output.elasticsearch:   hosts: ['${ELASTICSEARCH_HOST:elasticsearch}:${ELASTICSEARCH_PORT:9200}']   username: ${ELASTICSEARCH_USERNAME}   password: ${ELASTICSEARCH_PASSWORD}   ssl.verification_mode: none
YAML
Deploy
Let's deploy the Filebeat and check its status.
kubectl create -f filebeat-kubernetes.yaml kubectl --namespace=kube-system get ds/filebeat
Bash
