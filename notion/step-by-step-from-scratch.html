Metrics-Server
Download
Download Metrics-Server yaml and rename it
curl -LJO https://github.com/kubernetes-sigs/metrics-server/releases/download/v0.3.6/components.yaml mv components.yaml metrics-server-components.yaml
Bash
Settings
Open the metrics-server-components.yaml file in an editor and add the –kubelet-insecure-tls argument into the existing args section.
args: - --cert-dir=/tmp - --secure-port=4443 - --kubelet-insecure-tls
YAML
NOTE: DO NOT enable kubelet-insecure-tls on a cluster that will be accessed externally. This is only being done for a local Docker Desktop cluster.
Deploy
metrics-server-components.yaml
kubectl create -f metrics-server-components.yaml --save-config
Bash

Install step-issuer
Step Issuer is a cert-manager's CertificateRequest controller that uses step certificates (a.k.a. step-ca) to sign the certificate requests.
Step Issuer uses step certificates as the Certificate Authority or CA in charge of signing the CertificateRequest resources.
Installing step issuer
Download
First download the repo
https://github.com/smallstep/step-issuer
Bash
First we install the CRDs:
kubectl apply -f config/crd/bases
Bash
Then we install the controller:
kubectl apply -f config/samples/deployment.yaml
Bash
Adding a StepIssuer
Now, we're going to use all the configuration values that we got after installing step certificates and use them to configure our StepIssuer.
To get the CA URL
kubectl -n default logs job.batch/autocert
Bash
To get the base64 caBundle
kubectl get -o jsonpath="{.data['root_ca\.crt']}" configmaps/autocert-step-certificates-certs | base64
Bash
To get the kid
kubectl get -o jsonpath="{.data['ca\.json']}" configmaps/autocert-step-certificates-config

At this time Step Issuer is ready to sign certificates.
The YAML for a Certificate resource looks like:
apiVersion: cert-manager.io/v1alpha2 kind: Certificate metadata: name: backend-smallstep-com namespace: default spec: # The secret name to store the signed certificate secretName: backend-smallstep-com-tls # Common Name commonName: backend.smallstep.com # DNS SAN dnsNames: - localhost - backend.smallstep.com # IP Address SAN ipAddresses: - "127.0.0.1" # Duration of the certificate duration: 24h # Renew 8 hours before the certificate expiration renewBefore: 8h # The reference to the step issuer issuerRef: group: certmanager.step.sm kind: CertificateRequest name: step-issuer
YAML
To apply the certificate resource you just need to run:
kubectl apply -f certificate.yaml
Bash
Moments later a CertificateRequest will be automatically created by cert-manager:
kubectl get certificates.cert-manager.io
Bash
The Step Issuer gets this CertificateRequest and sends the sign request to step certificates, and stores the signed certificate in the same resource. Enable autocert (per namespace)
Using autocert is also easy:
•
Enable autocert for a namespace by labelling it with autocert.step.sm=enabled, then
•
Inject certificates into containers by annotating pods with autocert.step.sm/name: <name>
kubectl label namespace default autocert.step.sm=enabled
Bash
apiVersion: v1 kind: Namespace metadata: name: kubernetes-dashboard labels: autocert.step.sm: enabled
YAML
Annotate pods to get certificates
To get a certificate you need to tell autocert your workload's name using the autocert.step.sm/name annotation:
kind: Deployment apiVersion: apps/v1 metadata: labels: k8s-app: kubernetes-dashboard name: kubernetes-dashboard namespace: kubernetes-dashboard spec: replicas: 1 revisionHistoryLimit: 10 selector: matchLabels: k8s-app: kubernetes-dashboard template: metadata: annotations: # AUTOCERT ANNOTATION HERE -v ############################### autocert.step.sm/name: localhost # # AUTOCERT ANNOTATION HERE -^ ############################### labels: k8s-app: kubernetes-dashboard spec:
YAML
Now an X.509 (TLS/HTTPS) certificate is automatically created and mounted at /var/run/autocert.step.sm/ along with a corresponding private key and root certificate so we need to modify the args to pick up our certificate
containers: args: - --default-cert-dir=/var/run/autocert.step.sm - --tls-cert-file=site.crt - --tls-key-file=site.key - --auto-generate-certificates - --namespace=kubernetes-dashboard
YAML
--auto-generate-certificates can be left in place, and will be used as a fallback.
