we need the password to decrypt the provisioner private key, this is available in the secret step-certificates-provisioner-password, and can be obtained running:
kubectl get -o jsonpath='{.data.password}' secret/step-certificates-provisioner-password | base64 --decode
Bash
To get the kid
kubectl get -o jsonpath="{.data['ca\.json']}" configmaps/step-certificates-config
Bash
The root certificate can be obtained from the step-certificates-certs ConfigMap or running:
kubectl get -o jsonpath="{.data['root_ca\.crt']}" configmaps/step-certificates-certs
Bash
To configure the step issuer we will use the base64 version of the root certificate:
kubectl get -o jsonpath="{.data['root_ca\.crt']}" configmaps/step-certificates-certs | base64
Bash
With the previous helm install the CA URL will be https://step-certificates.default.svc.cluster.local.
helm install step-certificates smallstep/step-certificates
Bash
Connecting from outside the cluster
https://github.com/smallstep/autocert#connecting-from-outside-the-cluster
To connect we need a certificate. There are a couple different ways to get one, but for simplicity we'll just forward a port
kubectl port-forward service/step-certificates 4443:443
Bash
In another window we'll use step to grab the root certificate, generate a key pair, and get a certificate.
step ca bootstrap --fingerprint 65e846452e3d4282d121483ae86644bc1df305ec7b73c7a6fcb51212f36b90bb --ca-url 127.0.0.1:4443 step ca root root.crt --ca-url https://127.0.0.1:4443 --fingerprint 65e846452e3d4282d121483ae86644bc1df305ec7b73c7a6fcb51212f36b90bb
Bash
