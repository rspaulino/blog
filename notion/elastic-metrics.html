Running Metricbeat on Kubernetes
Let's get some data flowing in by deploying Metricbeat that supports capturing K8s metrics out of the box.
For Metricbeat to pick up K8 metrics it need kube-state-metrics to be deployed on your k8s cluster. Clone the repository, deploy kube-state-metrics, and confirm that it's deployed using the following commands.
git clone https://github.com/kubernetes/kube-state-metrics.git kube-state-metrics kubectl create -f kube-state-metrics/kubernetes watch kubectl get pods --namespace=kube-system | grep kube-state
Bash
Now download Metricbeat configuration file using below. We'll be modifying this file before using it to deploy Metricbeat on our Cluster.
curl -LJO https://raw.githubusercontent.com/elastic/beats/7.8/deploy/kubernetes/metricbeat-kubernetes.yaml
Bash
Everything is deployed under the kube-system namespace by default.
Settings
By default, Metricbeat sends events to an existing Elasticsearch deployment, if present. To specify a different destination, change the following parameters in the manifest file:
- name: ELASTICSEARCH_HOST value: elasticsearch - name: ELASTICSEARCH_PORT value: "9200" - name: ELASTICSEARCH_USERNAME value: elastic - name: ELASTICSEARCH_PASSWORD value: changeme
YAML
Once open, go to env: section (about line # 119 and 257) in the file. We will be changing ELASTICSEARCH_HOST and ELASTICSEARCH_PASSWORD values here.
Change ELASTICSEARCH_HOST value from elasticsearch to https://quickstart-es-http.default.svc.cluster.local and ELASTICSEARCH_PASSWORD with the one we will decoded.
kubectl get secret quickstart-es-elastic-user -o=jsonpath='{.data.elastic}' | base64 --decode; echo
