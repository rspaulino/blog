Enabling Metrics Server in Docker Desktop
Download:
curl -LJO https://github.com/kubernetes-sigs/metrics-server/releases/download/v0.3.6/components.yaml
Bash
components.yaml
Open the components.yaml file in an editor and add the –kubelet-insecure-tls argument into the existing args section.
args: - --cert-dir=/tmp - --secure-port=4443 - --kubelet-insecure-tls
YAML
NOTE: DO NOT enable kubelet-insecure-tls on a cluster that will be accessed externally. This is only being done for a local Docker Desktop cluster.
Deploy:
kubectl apply -f components.yaml
Bash
To see how things are going, first get the name of your Metrics Server Pod by running the following command:
kubectl get pods -n kube-system
Bash
Now run the following command and the logs should show it starting up and the API being exposed successfully:
kubectl logs [metrics-server-pod-name] -n kube-system
Bash
Give it a little time and you should now be able to run kubectl top commands!
kubectl top node kubectl top pods kubectl top pods --all-namespaces
Bash
HELM
https://github.com/helm/charts/tree/master/stable/metrics-server
helm install stable/metrics-server -n metrics --namespace kube-system --set args={--kubelet-insecure-tls}
Bash
