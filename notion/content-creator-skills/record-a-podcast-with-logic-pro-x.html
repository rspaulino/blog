Project Settings apply to an individual project.
When recording audio in Logic Pro, you might experience a slight delay between recording your voice and when you hear the sound from your speakers or headphones. This delay is called input monitoring latency. You can adjust this using the I/O Buffer Size.
Enables recording audio at 24-bit resolution. Keep this selected, unless storage space is an issue.

Setting up Logic Pro for Audio Recording
Setting Input and Output Devices
REMEMBER: You can tell Logic Pro which 'interface' or 'device' to use in Logic's Preferences
Open a new or existing project and navigate to Logic Pro X > Preferences > Audio
Creating a New Project
Uncheck Use musical grid, select the Interface or Device for input and output and Sample rate
We want an audio track for voice recording, here we select the input channel where the mic is connected and even select the Interface or Device
Leave Input Monitoring and Record Enable uncheck to avoid Feedback from speaker and mic at wrong levels. Before arming your tracks make sure you speaker levels aren't too high.
Feedback occurs when there is a loop between an audio input & output - like a microphone and speaker
Setting Up Recording Level
Select or switch the Input channel from the Interface where the mic is connected
To listen to your incoming audio - turn on 'Input Monitoring' on the track by clicking 'I'
To allow the track to be recorded on, 'record enable' the track by clicking 'R'
You can now start the project recording by clicking the Record button.
Before you hit record, begin talking into your mic as if you were recording. Watch the sound meter on the track’s control panel on the left. If you see the bars moving past yellow into orange and red, you are too loud and causing clipping. Use the control in the level meter to reduce your gain by moving the control to the left.

Recording with Effects
There's a common misconception that I come across a lot from people, it regards recording with effects specifically compression when you stick a compressor, an analog compressor in your signal path the signal that you're recording gets compressed before it reaches your audio interface. This is really the only effective true way to record with compression, the signal gets compressed and then gets printed to your digital audio file into your audio interface.
Now when you record with plugins on an input object in logic it's not the same thing because the compression is being applied after the signal hits your converters in audio interface, so it's no different than applying compression later after the file is already recorded. The point is to apply compression either before it reaches your audio interface and if not that then you may as well wait and not commit to printing with it and apply it later since there's no difference.
There may be situations where a performer wants to hear some effects in their headphones, let's say a voice over talent is recording and maybe they want a little bit of reverb in their headphones.
